# トレロ風タスク管理アプリ 要件定義書

## 1. 概要
ブラウザ上で動作する、Trelloに似たカンバン形式のタスク管理アプリを作成する。
学習目的のため、サーバーやビルド環境を用意せず、HTML/CSS/JavaScriptのみで動作させる。

## 2. 技術構成
- プレーンHTML / CSS / JavaScript（フレームワーク・ビルドツール不使用）
- データ保存は `localStorage` を使用し、ブラウザを閉じても・再起動してもデータを保持する

## 3. 画面構成
- ボード画面（メイン画面）
  - 複数の「リスト」を横並びで表示
  - 画面上部または末尾に「リストを追加」する入力欄

## 4. 機能要件

### 4.1 リスト
- リストの新規追加ができる（リスト名を入力して作成）
- リストの削除ができる
- リスト名のリネームができる
- ドラッグ＆ドロップによりリストの並び順を変更できる

### 4.2 カード
カードは以下の情報を持つ。
- テキスト（タスク名・必須）
- 期限日（任意）
- コメント（任意・複数件登録可能）

カードに対して以下の操作ができる。
- カードの新規追加（リスト単位）
- カードの削除
- カードの編集（テキスト・期限日・コメントの変更）
- ドラッグ＆ドロップによるリスト間の移動、および同一リスト内での並び替え

### 4.3 データ保存
- リスト・カードの全データは `localStorage` に保存する
- ページ再読み込み・ブラウザ再起動後も、保存したデータがそのまま復元される

## 5. データ構造（localStorage格納イメージ）

```json
{
  "lists": [
    {
      "id": "list-1",
      "title": "To Do",
      "cards": [
        {
          "id": "card-1",
          "text": "サンプルタスク",
          "dueDate": "2026-07-01",
          "comments": [
            { "id": "comment-1", "text": "メモ", "createdAt": "2026-06-27T00:00:00" }
          ]
        }
      ]
    }
  ]
}
```

リストの並び順は配列 `lists` の順序そのもので管理する（並び替え時はこの配列を入れ替えて保存）。

## 6. 非機能要件
- 対応環境: 最新のデスクトップブラウザ（Chrome/Edge等）
- レスポンシブ対応: 対象外（PC画面前提、必要なら後日拡張）
- 認証・複数ユーザー対応: 対象外

## 7. 今後の拡張候補（今回の対象外）
- サーバー・データベースへの保存（複数端末での共有）
- ラベル・担当者などの追加属性
- 検索・フィルタ機能

## 8. 開発の流れ
1. **フォルダ・ファイル準備**
   `index.html` / `style.css` / `script.js` の3ファイルを作成する
2. **HTMLで骨格を作る**
   ボード全体の枠、リスト1つ分の型（テンプレート）、カード1つ分の型を配置する。この時点ではまだ見た目は整っていない状態
3. **CSSで見た目を整える**
   リストを横並びに、カードをリストの中に縦並びに配置するカンバン風レイアウトを作る
4. **データの土台をJavaScriptで作る**
   - 「リスト一覧・カード一覧」をJavaScriptのデータ（配列）として持つ仕組みを作る
   - そのデータを画面に描画する関数を作る
   - localStorageへの保存・読み込み関数を作る
5. **リストの機能を実装**
   追加・削除・リネーム・並び替え（ドラッグ＆ドロップ）を、上記データ＋描画の仕組みに繋ぎ込む
6. **カードの機能を実装**
   追加・削除・編集（テキスト/期限日/コメント）・移動＆並び替え（ドラッグ＆ドロップ）を実装する
7. **動作確認**
   ブラウザで実際に操作し、保存→再読み込みでデータが残るか、各機能が要件通り動くかを確認する
